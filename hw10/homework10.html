Stochastic Process with Continuous Time

This page provides an interactive simulation of a stochastic process in continuous time, specifically a discrete approximation of a Poisson process defined on the interval [0, 1]. The process is constructed by dividing the time interval into n equally spaced steps and allowing events to occur randomly at each step according to a probability derived from the intensity parameter λ.

At every step, an event occurs with probability:
\( p = \frac{\lambda}{n} \)
and the cumulative number of events up to that time produces a sample path of the process. When multiple paths are simulated, they reveal the typical behaviour of a Poisson process, showing random fluctuations but also a clear statistical structure governed by λ.

The goal of this simulation is to visualize how a continuous–time stochastic process can emerge from many small random increments, and how its randomness becomes structured as parameters change. You can observe how different values of λ and different discretizations n influence the shape and variability of the resulting trajectories.

How to Use the Simulator

Attempts — Select how many independent realizations of the process you want to simulate. Each attempt produces one sample path.

λ (Lambda) — Controls the intensity of the process. A higher λ increases the expected number of events in the interval [0, 1]. At each discrete step, events occur with probability \( p = \lambda / n \).

n (Number of Steps) — Determines the granularity of the approximation. The interval [0, 1] is split into n steps of size \( 1/n \). Larger n values make the process behave more like an ideal continuous-time Poisson process.

After selecting parameters, the simulation automatically generates and displays:

  • the stochastic process sample paths over time;
  • the distribution of the final values reached at time 1.
By increasing the number of attempts, you can observe the emergent statistical patterns and the characteristic behaviour of a Poisson-driven process in continuous time.
